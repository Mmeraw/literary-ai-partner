Seed PR1 Contract

Scope: documentation and contract language only. No runtime route, no database migration, no Phase 2 trust path.

Existing governing artifact: accepted_story_ledger_v1.

Doctrine mapping: Golden Record is a plain-language doctrine term only. In code and persisted artifacts, the governing accepted ledger remains accepted_story_ledger_v1.

Seed rule: the seed may shape downstream work, but it may not authorize downstream truth.

Artifact level state describes the seed document lifecycle.
- created
- superseded
- archived
- failed

Claim level state describes each seeded hypothesis.
- proposed_unverified
- partially_confirmed
- confirmed_by_evidence
- drift_detected
- superseded_by_evidence
- invalidated

Allowed future seed artifacts:
- story_seed_v1
- evaluation_seed_v1

Both are non-governing scaffolds.

PR 1 acceptance criteria:
- docs only
- no runtime execution
- no Phase 2 routing changes
- no seed claim used as canon
- no new code term that replaces accepted_story_ledger_v1
- artifact state and claim state are separate
