## Purpose

Define the editorial layer that distinguishes **ritual repetition** from **mechanical repetition** so revision passes do not mistake incantatory force, mythic cadence, sensory recurrence, or escalating phrase structures for line-level error.

This canon exists because density-based literary horror generates authority through controlled recurrence, not maximum compression. Prior compression logic risked flattening texture, rhythm, and symbolic pressure when it treated all repetition as waste.

## Scope

Applies when a manuscript is tagged as literary-dense, ritual-bearing, atmosphere-led, mythic, horror, or any adjacent mode where voice depends on cadence and layered recurrence.

Governs:
- Line editing
- Compression logic
- Repetition evaluation
- Ritual phrase chains
- Symbolic recurrence
- AI revision prompts and human editorial override rules

## Core Doctrine

1. **Not all repetition is redundancy.** Repetition is protected when it deepens experience, alters pressure, or changes meaning across recurrence.
2. **Experience outranks mere efficiency.** If a line serves lived atmosphere, dread, compulsion, ritual force, or tonal spell-casting, inefficiency alone is not grounds for removal.
3. **Mechanical clarity is not the supreme value in literary horror mode.** Some passages must breathe, recur, echo, and accumulate to function correctly.
4. **Editorial intelligence must classify before cutting.** Every flagged line must first be identified as mechanical, explanatory, weak-authority, flat repetition, or ritual repetition before any action is taken.

## Governing Question

All line-level decisions begin with one diagnostic split:
- **Is this moment about clarity?** -> Eligible for cleaning
- **Is this moment about experience?** -> Must be allowed to breathe unless it fails ritual standards

## Decision Tree

| Line Type | Required Action |
|---|---|
| Mechanical artifact | CUT |
| Explanatory redundancy | CUT |
| Weak authority line (good idea, thin execution) | REWRITE in voice |
| Flat repetition with no escalation | REDUCE |
| Ritual repetition that adds force, variation, or pressure | PRESERVE |

## Ritual Test

A repeated line, fragment, image, or pattern qualifies as **ritual** when:
- It adds a new image or sensation
- It escalates intensity, panic, compulsion, possession, dread, or reverence
- It repeats with variation in meaning rather than merely restating the same beat
- It increases cadence, spell effect, incantatory pull, or thematic charge

A repeated element **fails** the ritual test when:
- It repeats with no change
- It repeats weaker than the prior instance
- It explains what the prior line already made the reader feel

## Canon Table

| Question | If Yes | If No |
|---|---|---|
| Does it add a new image or sensation? | KEEP | Continue test |

## Protected Patterns

Presumptively protected unless they clearly fail the ritual test:
- Short-fragment escalation chains
- Repeated imperative pulses (command language)
- Symbolic object recurrence
- Liturgical or prayer-like phrase structures
- Sensory returns in smell, sound, vibration, texture, or environmental pressure
- Repeated triads that intensify rather than idle

## Forbidden Editorial Errors

The editor must NOT:
- Cut a repeated line only because it appears more than once
- Collapse ritual cadence into plain exposition
- Remove sensory recurrence that is building atmosphere or compulsion
- Treat mythic echo as duplicate information by default
- Optimize prose until it becomes too clean, too efficient, too controlled, or tonally sterilized

## Pass/Fail Rule

A revision pass is **non-compliant** with RITUAL-EDITOR-1 if it:
- Cuts lines that add new image, sensation, or escalation
- Breaks ritual cadence chains that are functioning
- Replaces mythic or experiential recurrence with explanatory smoothing
- Produces prose that is cleaner but less haunted, less atmospheric, or less authoritative

Compliant only when it removes true comfort-repeat, mechanical redundancy, and weak authority language while preserving the spell system of the prose.

## Operational Summary

RITUAL-EDITOR-1 exists to stop the system from confusing **alive repetition** with **dead repetition**. It authorizes cutting what is merely repetitive, but it protects recurrence that behaves like ritual, pressure, spell, compulsion, theology, or dread.

It is the governing layer above WAVE-55-L: WAVE-55-L decides what to compress, while RITUAL-EDITOR-1 decides what repetition is sacred enough to leave untouched.
