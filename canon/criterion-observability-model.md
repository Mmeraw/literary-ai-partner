# Criterion Observability & Signal Sufficiency Model

> Canon Level: MANDATORY  
> Enforcement: Synthesis boundary + Quality Gate + Finalizer + Renderer

## Governing law

**Applicability is determined by system governance. Observability is determined by manuscript evidence.**

**Evaluation is constrained by evidence, not schema.**

---

## Criterion state model (canonical)

```ts
type CriterionStatus =
  | "NOT_APPLICABLE"      // governed only (criteria_plan / MDM / eval mode)
  | "NO_SIGNAL"
  | "INSUFFICIENT_SIGNAL"
  | "SCORABLE";
```

| Status | Origin | score_0_10 | Meaning |
|---|---|---:|---|
| `NOT_APPLICABLE` | Governed | `null` | Criterion does not apply for this evaluation context |
| `NO_SIGNAL` | Observed | `null` | No relevant evidence found |
| `INSUFFICIENT_SIGNAL` | Observed | `null` | Evidence exists but not enough to score |
| `SCORABLE` | Observed | `0-10` integer | Sufficient evidence to evaluate |

### Critical invariants

- `score_0_10` exists **iff** `status === "SCORABLE"`.
- `NOT_APPLICABLE` may **never** be model-invented.
- `NOT_APPLICABLE` must be sourced from governance (`criteria_plan`, MDM matrix, or evaluation mode).
- `NO_SIGNAL` / `INSUFFICIENT_SIGNAL` must carry structured insufficiency reason.

---

## Structured insufficiency reason

When non-scorable due to evidence state, reason must be structured:

```json
{
  "looked_for": ["..."],
  "not_found": ["..."],
  "evidence_span": {
    "char_start": 0,
    "char_end": 0
  }
}
```

Free-text-only reasons are non-canonical.

---

## Completeness bridge rule

A criterion is **complete** if it satisfies one of:

### A) Scorable
- `status = SCORABLE`
- `scorable = true`
- `score_0_10` is integer `0..10`
- rationale present
- anchor sufficiency met

### B) Non-scorable by evidence
- `status in { NO_SIGNAL, INSUFFICIENT_SIGNAL }`
- `scorable = false`
- `score_0_10 = null`
- rationale present
- structured insufficiency reason present

### C) Non-applicable by governance
- `status = NOT_APPLICABLE`
- `scorable = false`
- `score_0_10 = null`
- no insufficiency reason required

### Invalid
- criterion missing from canonical 13
- duplicate criterion key
- score present for non-scorable status
- null score for `SCORABLE`
- `NOT_APPLICABLE` generated by model output

---

## Deterministic evidence rules

Anchors are not distinct if:
- same snippet/sentence,
- char-span overlap > 50%,
- within same 200-char window and lexical overlap > 60%.

Pattern criteria (`voice`, `tone`, `pacing`, `theme`) require distributed anchors across separated text regions; adjacent micro-phrases do not satisfy pattern distribution.

Prose Control threshold is locked: **5+ sentences OR >=30% passage coverage** for scoreability.

---

## Aggregation and rendering rules

- Overall score is computed from `SCORABLE` criteria only.
- If `scored_criteria_count = 0`, overall score is `null` (never `0`).
- Renderer must never display `0/10` for non-scorable states.

| State | Renderer label |
|---|---|
| `NOT_APPLICABLE` | `N/A — Not applicable for this evaluation context` |
| `NO_SIGNAL` | `N/A — No observable evidence` |
| `INSUFFICIENT_SIGNAL` | `N/A — Insufficient manuscript evidence` |
| `SCORABLE` | `{score} / 10` |

Renderer must disclose denominator:

> Overall score based on X of 13 scorable criteria.
