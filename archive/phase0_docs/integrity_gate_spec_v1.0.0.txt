Canon Spec — Evaluation Integrity Gate (v1.0.0)
Status: CANON-READY (binding)
Scope: Evaluation pipeline (Quick + Full), post-LLM, pre-render, pre-persist
Purpose: Prevent “pretty lies” by enforcing a single source of truth for criteria applicability, scoring, aggregation, and output rendering.

0. Definitions
- criteriaPlan: The authoritative plan generated by policy routing for a given workTypeUi + sample class (e.g., S1 micro).
- CriterionStatus: "SCORED" | "NOT_ASSESSED"
- scoredCriteria: criteriaPlan items where status == "SCORED"
- naCriteria: criteriaPlan items where status == "NOT_ASSESSED"
- scored_count: Count of criteria that contain a numeric score in the LLM result payload.
- expected_scored: scoredCriteria.length (must never be hard-coded).
- craftScore: numeric aggregate score computed ONLY from scoredCriteria.

1. Canon Rule — Single Source of Truth (Non-Negotiable)
criteriaPlan is the sole authority for:
- which criteria are scored
- which criteria are "Not assessed"
- expected_scored
- craftScore denominator

No other component may:
- assume 13 criteria
- infer applicability from text content ("no dialogue" etc.)
- compute craftScore using total criteria (scored + N/A)

2. Required Inputs
Integrity Gate MUST be called with:
- criteriaPlan (final, post-enforcement)
- llmResult (raw structured output)
- workTypeUi + sampleClass (metadata, for logging only)

3. Required Outputs (GateSummary)
The gate returns:
- pass: boolean
- expected_scored: number
- scored_count: number
- na_criteria: array<{ key, reason }>
- failures: array<{ code, message, details? }>
- craft_score_denominator: number
- craft_score_numerator: number | null
- craft_score: number | null

4. Hard Fail Conditions (must fail closed)
4.1 SCORED_COUNT_MISMATCH
If scored_count != expected_scored => FAIL.

4.2 NA_CRITERIA_MISSING
If naCriteria.length > 0 AND output.na_criteria is empty or missing => FAIL.

4.3 NA_CRITERIA_CONTAMINATION
If any criterion marked NOT_ASSESSED contains a numeric score in llmResult => FAIL.

4.4 MISSING_REQUIRED_SCORES
If any criterion marked SCORED is missing a numeric score => FAIL.

4.5 CRAFT_SCORE_DENOMINATOR_SUSPECT
If craftScore is computed with denominator != expected_scored => FAIL.
If craftScore exists but expected_scored == 0 => FAIL.

5. Soft Fail Conditions (warn, do not block)
- Missing optional sections that do not affect truth (e.g., supporting perspectives)
(Soft fails are allowed only if they cannot alter scoring truth.)

6. Rendering Contract (binding)
If naCriteria.length > 0, the UI/output MUST include a visible "Not Assessed" section listing each NA criterion and its reason.
If naCriteria.length == 0, the section may be omitted.

7. Logging / Telemetry (required)
On FAIL, emit:
- EVAL_INTEGRITY_FAILURE
- GateSummary.failures codes
- expected_scored, scored_count
- naCriteria keys
- workTypeUi, sampleClass, evaluationRunId/requestId
No raw manuscript text is required in logs.

8. Test Contract (must exist)
A golden regression test MUST verify:
- Micro/S1 plans produce <= policy max scored criteria
- Not Assessed criteria appear in output
- craftScore denominator equals expected_scored
- Any drift triggers FAIL

9. Compatibility / Migration
Legacy pipelines must either:
(A) adopt criteriaPlan-based gating, or
(B) set pass=false and return a governed “Evaluation unavailable” response.
No partial compliance is allowed.

End.
