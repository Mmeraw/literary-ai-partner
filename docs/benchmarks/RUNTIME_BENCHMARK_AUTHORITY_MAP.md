## Runtime use by phase

- Phase 0: load this map, hash/checksum listed files where required, and persist the loaded path list in `phase0_authority_proof_v1`.
- Phase 0.5A: use the map to ground `story_map_seed_v1` and full-context Story Ledger seed generation.
- Phase 0.5B: use the map to ground `revise_opportunity_seed_v1` and opportunity pattern selection.
- Pass 1 / Pass 2: use the map as calibration background only; manuscript evidence remains primary.
- Pass 3A / Pass 3B: use the map to enforce DREAM long-form multi-layer structure, governed ledgers, cross-layer synthesis, and benchmark-quality report completeness.
- Quality Gate / Semantic Gate: use the map to identify missing layers, weak structural coverage, unsupported benchmark drift, and stale output-mode language.
- WAVE: use the map to compare revision priorities against current DREAM expectations.
- Revise Queue / Revise Workbench: use the map for opportunity-shape and layer-completeness standards, but never as manuscript evidence.

## Hard rule

Benchmarks define target quality and structural expectations. They do not replace manuscript evidence, do not supply facts about a submitted manuscript, and do not override the accepted Story Ledger.
