# U1.1 — Wire deriveConfidence() into finalizer and persist to artifact

Closes #158

## Purpose
Connect the pure U1 derivation module (#157) into the live evaluation finalizer so every completed job writes a canonical `confidence` + `confidence_reasons` pair to its artifact. Wiring-only follow-up. No U1 logic changes. No new confidence states. No UI.

## Scope

### In scope
- Call `deriveConfidence(inputs)` from the finalizer (or report-builder) after evaluation assembly, before artifact release.
- Build `ConfidenceInputs` from signals already available at finalizer scope.
- Persist result to the final artifact (new JSON fields, not a new column), including `confidence_derivation_version`.
- Integration tests that assert confidence given known job states.

### Out of scope
- UI rendering of confidence (separate PR).
- Schema migrations / new columns.
- Changes to U1 derivation logic.

## Acceptance criteria
- [ ] `deriveConfidence()` invoked exactly once per finalized job, pre-release.
- [ ] Artifact JSON contains `confidence`, `confidence_reasons[]`, `confidence_derivation_version`.
- [ ] Integration tests cover: happy-path, missing-signal, validity_status=null path.
- [ ] No changes to `lib/u1/derive.ts` (pure module untouched).
- [ ] CI green (lint, typecheck, unit, integration).

## Test plan
1. Unit: finalizer builds `ConfidenceInputs` correctly from a fixture job.
2. Integration: end-to-end job finalization writes expected artifact shape.
3. Regression: existing finalizer tests still pass.
