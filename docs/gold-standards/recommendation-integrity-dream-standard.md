# Recommendation Integrity Dream Standard

## Purpose

This document defines the quality standard for all author-facing recommendations produced by RevisionGrade™, including:

- Evaluation Report recommendations
- Revision Opportunity Ledger entries
- Revise Queue cards
- TrustedPath recommendations
- Future Storygate editorial recommendations

This standard exists because a recommendation can be:

- Grammatically valid
- Structurally complete
- Editorially useless

RevisionGrade must reject recommendations that satisfy only the first two conditions.

**The objective is not merely valid recommendations.**

**The objective is recommendations that a sophisticated developmental editor would be willing to sign their name to.**

## Core Principle

A recommendation must do more than identify a problem.

A recommendation must:

1. identify the problem,
2. explain why it matters,
3. locate it in the manuscript,
4. propose a specific repair,
5. explain the expected reader impact.

If any of these are absent, the recommendation is incomplete.

## Four Quality Tiers

### FAIL

The recommendation is unsuitable for author consumption.

Examples:

- malformed text
- sentence fragments
- missing fields
- orphan conjunctions
- missing causality
- generic workshop language
- missing anchors
- unclear reader effect

Example:

> Insert a concrete stakes beat.

Status: **FAIL**

Reason:

- No anchor
- No scene reference
- No explanation
- No reader outcome

### PASS_MINIMUM

The recommendation is technically valid but lacks sufficient specificity.

Example:

> Insert a visible consequence after the decision to improve momentum.

Status: **PASS_MINIMUM**

Strengths:

- grammatical
- complete
- causal

Weaknesses:

- generic
- not anchored
- weak editorial value

Evaluation reports may tolerate PASS_MINIMUM recommendations.

Revise Queue should not.

### PASS_STRONG

The recommendation is specific, useful, and actionable.

Example:

> After Nicolas decides to return to British Columbia, add a visible consequence showing what Mike believes has failed about the Toronto experiment. This clarifies the emotional cost of the decision and strengthens momentum into the next section.

Status: **PASS_STRONG**

Strengths:

- anchored
- specific
- actionable
- explains reader impact

Suitable for:

- Evaluation Reports
- Revise Queue

### PASS_DREAM_STANDARD

The recommendation demonstrates editorial intelligence.

Example:

> After Nicolas tells Mike he is homesick, add a visible consequence that forces Mike to decide whether Toronto has failed as a rescue attempt. The current version moves directly to the return flight, which reports the outcome without dramatizing the emotional cost. By requiring Mike to confront the failure before acting, the scene converts reflection into conflict and strengthens the manuscript's recurring question about whether love can meaningfully change another person's trajectory.

Status: **PASS_DREAM_STANDARD**

Strengths:

- anchored
- scene-specific
- character-specific
- causally grounded
- emotionally grounded
- thematically grounded
- reader effect explained
- developmental-editor quality

This is the target standard for RevisionGrade.

## Mandatory Components

Every recommendation must contain:

### Symptom

What is wrong?

Must reference manuscript evidence.

Acceptable:

> The current scene moves directly from Nicolas announcing he is homesick to the return flight.

Unacceptable:

> The scene lacks momentum.

### Cause

Why is it happening?

Must contain causal reasoning.

Acceptable:

> Because the emotional decision occurs off-page, readers observe the outcome without experiencing the conflict.

Unacceptable:

> The scene feels weak.

### Fix Direction

What should change?

Must describe an editorial action.

Acceptable:

> Add a decision beat before the return flight.

Unacceptable:

> Improve the scene.

### Reader Effect

Why does the repair matter?

Acceptable:

> This increases emotional investment by allowing readers to experience the failure rather than being told about it afterward.

Unacceptable:

> Improves engagement.

### Anchor

Where should the author look?

Acceptable:

> Near "I flew him home three days later."

Unacceptable:

> In the passage.

## Dream Standard Requirements

To achieve PASS_DREAM_STANDARD, a recommendation should contain:

- Character name
- Scene reference
- Quoted anchor
- Decision point
- Consequence
- Thematic connection
- Reader outcome

The more of these present, the stronger the recommendation.

## Generic Workshop Language Ban

The following phrases are not sufficient by themselves:

- increase engagement
- improve clarity
- strengthen pacing
- raise tension
- improve immersion
- enhance impact
- improve flow
- deepen characterization

These phrases may appear only if accompanied by concrete manuscript evidence.

## Integrity Gate Policy

All author-facing recommendations must pass the Recommendation Integrity Gate.

Failure behavior:

1. Detect violation
2. Generate structured failure code
3. Attempt one regeneration
4. Re-evaluate
5. If still failing:
   - quarantine
   - admin visible only
   - never shown to author

RevisionGrade shall prefer:

> 8 excellent recommendations

over

> 14 recommendations containing one weak or malformed recommendation.

Author trust is more important than recommendation count.

## Governing Principle

The recommendation is the product.

Authors judge RevisionGrade less by its diagnosis than by the quality of the actions it recommends.

A recommendation should survive scrutiny from:

- professional editors
- literary agents
- acquisition editors
- experienced authors
- skeptical readers

If the recommendation would embarrass a competent developmental editor, it does not meet the RevisionGrade Dream Standard.
