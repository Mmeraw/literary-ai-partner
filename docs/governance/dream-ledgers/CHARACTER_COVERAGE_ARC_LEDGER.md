# DREAM Character Coverage & Arc Ledger

## Purpose

The Character Coverage & Arc Ledger ensures that a DREAM long-form report accounts for the manuscript's major character system with the same seriousness applied to plot, theme, structure, and closure.

This is a report-quality and trust layer. A report may sound fluent and still fail if it omits a co-protagonist, underweights a recurring companion, contradicts pronouns, or ignores a character whose ending obligation shapes reader payoff.

## Required when

Required for every DREAM long-form report.

## Folded report surfaces

This ledger should be rendered through existing DREAM surfaces, especially:

- `structural_stack`
- `layer_analyses`
- `criterion_analyses` for character, POV/voice, closure, and marketability
- `reader_experience`
- `acceptance_checks.required_detection`
- `acceptance_checks.failure_conditions`

## Minimum row contract

Each major row should account for:

| Field | Requirement |
|---|---|
| Character | Canonical name or label. |
| Aliases / renamings | Required when identity, naming, or aliases matter. |
| Role | Protagonist, co-protagonist, antagonist, major secondary, recurring companion, animal companion, symbolic force, collective force, or other. |
| Narrative weight | Primary, major, supporting, recurring, or minor. |
| Relationship engines | Principal relationships the character participates in. |
| Arc start | Initial dramatic/emotional/ethical state. |
| Arc pressure | Main pressure applied to the character. |
| Turning movement | Major turn, reversal, revelation, choice, or pressure shift. |
| End state | Final state or current ending condition. |
| Ending accountability | Resolved, transformed, tragically confirmed, intentionally unresolved, underpaid, accidentally abandoned, or missing from ending. |
| Evidence anchors | Specific scenes, chapters, objects, behaviors, or recurring signals supporting the row. |
| Report risk | Covered, underweighted, omitted, or needs manual verification. |

## Hard-fail conditions

A DREAM report is incomplete if any of the following occurs:

- A protagonist is missing from the ledger or rendered report.
- A co-protagonist is omitted from the rendered report.
- A major recurring companion is omitted or materially underweighted.
- A major character has no arc-state coverage.
- A major character has no ending-status accountability.
- The report contradicts a major character's pronouns or gender.
- The report discusses closure while omitting a principal character's payoff obligation.

## Soft-fail / rerender conditions

- A major character is present but generically classified.
- Arc coverage names traits but not movement from start → pressure → turn → end state.
- Ending status is assigned without evidence.
- Relationship closure is weaker than individual character closure coverage.
- A visible report section and the structured ledger diverge in emphasis.
