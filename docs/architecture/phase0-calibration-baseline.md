# Phase 0 Calibration Baseline Architecture

Status: Canonical architecture definition  
Created: 2026-06-09  
Purpose: Define the precomputed `phase0_calibration_baseline_v1` artifact that replaces per-job governance doc re-interpretation.

---

## Problem Statement

Every evaluation job was previously re-reading and re-interpreting the same static governance documents (Dream standards, Gold standards, Canon, fail-closed rules, what-not-to-do). This:

- Added unnecessary latency (LLM calls to summarize the same docs every time)
- Risked drift between interpretations across jobs
- Violated the 12–15s Phase 0 SLA
- Conflated "understanding governance" with "binding authority to a job"

## Architecture Decision

**Phase 0 is authority binding, not live document interpretation.**

A certified, precomputed calibration baseline is generated offline (admin/build-time) from static governance documents. Runtime Phase 0 loads this baseline, verifies its integrity, and binds it to the current job.

## `phase0_calibration_baseline_v1` Schema

```typescript
interface Phase0CalibrationBaseline {
  /** Artifact identity */
  artifact_type: "phase0_calibration_baseline_v1";
  version: string;          // Semantic version (e.g., "1.0.0")
  checksum: string;         // SHA-256 of the serialized content
  generated_at: string;     // ISO 8601 timestamp
  generated_by: string;     // "admin" | "build-system"

  /** Source document manifest */
  source_documents: {
    path: string;           // Relative path in repo
    checksum: string;       // SHA-256 of the source doc at generation time
    version?: string;       // Optional version tag
  }[];

  /** Calibration content — what the system "knows" before touching any manuscript */
  calibration: {
    /** What RevisionGrade is trying to achieve */
    evaluation_posture: string;

    /** What excellence looks like — distilled from Dream standards */
    excellence_model: {
      summary: string;
      key_behaviors: string[];
      dream_standard_refs: string[];
    };

    /** What failure looks like — distilled from Gold standards + fail-closed rules */
    failure_model: {
      summary: string;
      failure_modes: string[];
      gold_standard_refs: string[];
    };

    /** What must never be done — from WHAT_NOT_TO_DO.md */
    prohibited_behaviors: string[];

    /** Canon hierarchy for authority resolution */
    canon_hierarchy: string[];

    /** 13 evaluation criteria definitions */
    criteria_definitions: {
      criterion_id: string;
      name: string;
      definition: string;
      excellence_signal: string;
      failure_signal: string;
    }[];

    /** Confidence boundaries and routing rules */
    routing_rules: {
      short_form_threshold: number;     // word count
      long_form_threshold: number;      // word count
      multi_layer_threshold: number;    // word count
    };

    /** Revise Queue quality bar */
    revise_quality_bar: {
      minimum_candidate_requirements: string[];
      prose_contract: string;
    };
  };
}
```

## Generation Process (Offline / Admin)

The calibration baseline is generated by:

1. Loading all governance source documents (Dream standards, Gold standards, Canon, fail-closed rules, what-not-to-do, SIPOC)
2. Using an LLM to synthesize the calibration content (one-time cost)
3. Human review + approval of the synthesized content
4. Checksumming and versioning the result
5. Persisting as a static artifact (file or database record)

**Generation is NOT part of the evaluation pipeline.** It happens:
- When governance docs change
- When Dream/Gold standards are updated
- On admin/operator command
- As part of release/deploy process

## Runtime Phase 0 Contract

```
Input:  phase0_calibration_baseline_v1 (precomputed, versioned)
        + manuscript metadata (word count, structure)

Process:
  1. Load calibration baseline
  2. Verify checksum matches expected version
  3. Confirm all source document checksums still valid
  4. Select route based on manuscript metadata + routing rules
  5. Persist phase0_authority_proof_v1

Output: phase0_authority_proof_v1
        {
          baseline_version: string,
          baseline_checksum: string,
          authority_paths_confirmed: boolean,
          selected_route: "short_form" | "long_form" | "long_form_multilayer",
          route_rationale: string,
          manuscript_word_count: number,
          timestamp: string
        }

SLA:    12–15 seconds target, 20 seconds hard limit
LLM:    NONE — purely deterministic
```

## Relationship to Other Stages

```
Phase 0 (Authority Binding)
  └── Loads phase0_calibration_baseline_v1
  └── Outputs phase0_authority_proof_v1
        │
        ├── Phase 0.5A (Story Seeds)
        │     └── Uses baseline + manuscript → story_map_seed_v1 + evaluation_seed_v1
        │     └── Minimum 500 words combined seed output
        │
        ├── Phase 0.5A (Enhanced Ledger)
        │     └── Uses seeds + manuscript → full_context_story_ledger_v1
        │     └── SLA: scope-dependent, 180s hard limit
        │
        └── Phase 0.5B (Revise Opportunity Seed)
              └── Uses baseline + manuscript + 13 criteria → editorial_dream_seed_v1
```

## Migration Path

**Current state:** Runtime re-reads governance docs per job (slow, redundant, drift-prone).

**Target state:** Precomputed baseline loaded deterministically per job (fast, consistent, verifiable).

**Migration steps:**
1. ✅ Document architecture (this file)
2. ⬜ Generate initial `phase0_calibration_baseline_v1` from current governance docs
3. ⬜ Implement baseline loader in `phase0AuthorityProof.ts`
4. ⬜ Implement checksum verification
5. ⬜ Remove per-job governance doc re-interpretation from runtime
6. ⬜ Add baseline regeneration command (admin CLI or build step)
7. ⬜ Validate Phase 0 meets 12–15s SLA with baseline approach

---

## Versioning Policy

- Baseline version increments on ANY governance doc change
- Old baselines remain available for audit trail
- Jobs record which baseline version they used in `phase0_authority_proof_v1`
- If source doc checksums change, baseline is automatically invalidated (fail-closed)
